# Кадровый дашборд

Репозиторий для кадрового мониторинга: еженедельные выгрузки от кадров, рабочий реестр, офлайн HTML-дашборд, списки по статусам и история изменений.

## Цель

Собрать устойчивый процесс:

1. Кадры присылают еженедельную выгрузку в формате файла от 08.07.2026.
2. Выгрузка добавляется в проект как исходный файл.
3. Данные нормализуются в рабочий реестр.
4. Обновляется дашборд и списки: 0,5 ставки, сокращение, вакансии, закрепления.
5. Все изменения фиксируются в GitHub: дата, суть правки, причина, итоговые цифры.

## Что уже заложено в MVP

- дашборд с KPI;
- фильтры по статусам;
- список 0,5 ставок;
- список сокращения с фамилиями;
- список вакансий;
- группировка по замам/блокам;
- группировка по закреплённым;
- карточка сотрудника;
- пояснение по разнице между строками/людьми и ставками.

## Важные рабочие правила

- `ВАКАНТ_Анашкин` учитываем как блок `Дивин`.
- Максаев Максим Вадимович относится к блоку `Дивин`, не к `Попов`.
- У Викуловой по текущему знанию 6 сотрудников на 0,5 ставки.
- По сокращению считаем отдельно строки/людей и ставки, потому что 0,5 ставки искажают простую численность.
- Сырой файл кадров не редактируем руками.
- Любая существенная правка фиксируется в `CHANGELOG.md`.

## Структура

```text
00_instructions/       правила работы и структура данных
01_raw_uploads/        исходные еженедельные выгрузки кадров
02_working_files/      рабочие Excel/Google Sheets шаблоны
03_offline_html/       офлайн HTML-дашборды для показа
04_exports/            выгрузки по замам, списки, отчеты
05_archive/            архив старых версий
```

## Как показывать руководству

Открыть файл из папки `03_offline_html/` в браузере. Он работает офлайн, без интернета и без сервера.
