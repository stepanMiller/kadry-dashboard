# Еженедельный процесс обновления

## 1. Получить файл от кадров

Кадры присылают очередную выгрузку в формате файла от 08.07.2026.

## 2. Сохранить исходник

Сырой файл не редактируется руками. Он кладётся в папку:

```text
01_raw_uploads/YYYY-MM-DD/
```

## 3. Обновить рабочий реестр

Из исходника и управленческих справочников формируется рабочий реестр:

- ФИО сотрудника;
- зам/блок;
- ФИО заместителя;
- фактическое подразделение;
- актуальная должность;
- статус;
- ставка;
- закреплён за;
- примечание;
- дата изменения.

## 4. Проверить изменения

Сравнить текущую неделю с предыдущей:

- новые сотрудники;
- исчезли из выгрузки;
- изменилась ставка;
- изменился статус;
- изменился зам/блок;
- изменилось подразделение;
- появились пустые или спорные поля.

## 5. Обновить дашборд

Обновить:

- KPI;
- списки 0,5 ставок;
- сокращение с фамилиями;
- вакансии;
- группировку по замам;
- группировку по закреплённым;
- карточку сотрудника.

## 6. Зафиксировать в GitHub

Каждая существенная правка должна иметь понятный commit message и запись в `CHANGELOG.md`.
